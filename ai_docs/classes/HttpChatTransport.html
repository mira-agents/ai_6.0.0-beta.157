      Sends messages to the chat API endpoint and returns a streaming response.

      This method handles both new message submission and message regeneration. It supports real-time streaming of responses through UIMessageChunk events.

      Parameters

      • options: {
            trigger: "submit-message" | "regenerate-message";
            chatId: string;
            messageId: string | undefined;
            messages: UI_MESSAGE[];
            abortSignal: AbortSignal | undefined;
        } & ChatRequestOptions

        Configuration object containing:

        • trigger: "submit-message" | "regenerate-message"

          The type of message submission - either new message or regeneration

        • chatId: string

          Unique identifier for the chat session

        • messageId: string | undefined

          ID of the message to regenerate, or undefined for new messages

        • messages: UI_MESSAGE[]

          Array of UI messages representing the conversation history

        • abortSignal: AbortSignal | undefined

          Signal to abort the request if needed

        • Optionalheaders?: Record<string, string> | Headers

          Additional headers that should be to be passed to the API endpoint.

        • Optionalbody?: object

          Additional body JSON properties that should be sent to the API endpoint.

        • Optionalmetadata?: unknown

      Returns Promise<ReadableStream<UIMessageChunk>>

      Promise resolving to a ReadableStream of UIMessageChunk objects. The stream emits various chunk types like:

      • text-start, text-delta, text-end: For streaming text content
      • tool-input-start, tool-input-delta, tool-input-available: For tool calls
      • data-part-start, data-part-delta, data-part-available: For data parts
      • error: For error handling

      Throws

      Error when the API request fails or response is invalid

    reconnectToStream

    • reconnectToStream(
          options: { chatId: string } & ChatRequestOptions,
      ): Promise<ReadableStream<UIMessageChunk> | null>

      Reconnects to an existing streaming response for the specified chat session.

      This method is used to resume streaming when a connection is interrupted or when resuming a chat session. It's particularly useful for maintaining continuity in long-running conversations or recovering from network issues.

      Parameters

      • options: { chatId: string } & ChatRequestOptions

        Configuration object containing:

        • chatId: string

          Unique identifier for the chat session to reconnect to

        • Optionalheaders?: Record<string, string> | Headers

          Additional headers that should be to be passed to the API endpoint.

        • Optionalbody?: object

          Additional body JSON properties that should be sent to the API endpoint.

        • Optionalmetadata?: unknown

      Returns Promise<ReadableStream<UIMessageChunk> | null>

      Promise resolving to:

      • ReadableStream<UIMessageChunk>: If an active stream is found and can be resumed
      • null: If no active stream exists for the specified chat session (e.g., response already completed)

      Throws

      Error when the reconnection request fails or response is invalid
